CodePilot AI v4 - Implementation Plan

Vision

Transform CodePilot AI from a feature-rich engineering demo into a
premium AI-powered Repository Intelligence platform.

Core Product Statement: “CodePilot helps developers understand
unfamiliar codebases with AI.”

This redesign focuses on: - Simplicity - Workflow-first UX - Progressive
disclosure - Premium UI - Repository understanding

Do NOT redesign around features. Design around developer goals.

  ------------------
  CURRENT PROBLEMS
  ------------------

-   Too many top-level features.
-   Navigation exposes implementation details.
-   No clear primary workflow.
-   Editor is not the main focus.
-   Two competing dashboards.
-   Right sidebar is overloaded.
-   Too many borders/cards.
-   Empty states feel unfinished.

  ---------------------
  TECHNICAL DECISIONS
  ---------------------

Frontend: - React - JSX - Vite - Tailwind CSS - Zustand - Monaco
Editor - React Flow

Backend: - FastAPI - Python - OpenRouter - Docker

Decision: Stay on JSX. Do NOT migrate to TypeScript during v4. Migrate
only after the product architecture stabilizes.

Recommended libraries: - Framer Motion - cmdk - TanStack Virtual -
Lucide React - shadcn/ui (selectively)

  ------------------------------------
  PHASE 1 — INFORMATION ARCHITECTURE
  ------------------------------------

Replace feature-driven navigation with workflow-driven navigation.

Primary Navigation

Repository Search Editor Assistant

Everything else becomes contextual.

Repository - Overview - Explorer - Recent Files - Bookmarks - Statistics

Search - Semantic Search - File Search - Symbol Search - AI Search

Editor - Monaco - Tabs - Diff View - Problems

Assistant - Chat - AI Actions - History - Tasks

Remove from top-level navigation: - Planner - Workspace - Patch -
Telemetry - Knowledge Graph - Analytics

  ------------------
  SUCCESS CRITERIA
  ------------------

The redesign is complete when:

-   Users understand the product within 30 seconds.
-   Repository Overview is the default landing page.
-   Editor dominates the interface.
-   Navigation reflects workflows, not implementation.
-   Features are progressively disclosed.
-   The UI feels cohesive, premium, and purpose-built for repository
    intelligence.

FINAL PRINCIPLE

Do not try to become Sourcegraph.

Sourcegraph helps developers FIND code.

CodePilot should help developers UNDERSTAND code.

That is the product identity.
